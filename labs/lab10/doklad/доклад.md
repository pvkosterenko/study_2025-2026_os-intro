# Доклад по теме "Сеть в UNIX. Обзор стратегий маршрутизации и маршрутизаторов"

## Введение

Сетевые технологии являются неотъемлемой частью любой современной операционной системы, и Linux не исключение. Будучи одной из самых популярных серверных платформ, Linux предоставляет администраторам и инженерам гибкие и мощные инструменты для управления маршрутизацией — процессом выбора пути для передачи пакетов данных между сетями.

В данном докладе мы подробно рассмотрим, какие стратегии маршрутизации доступны в UNIX-подобных системах, как они реализуются с помощью стандартных утилит Linux, а также каким образом обычный компьютер с Linux может быть превращён в полноценный сетевой маршрутизатор, способный конкурировать с дорогостоящим аппаратным обеспечением.

Маршрутизация в Linux — это не просто одна таблица со шлюзом по умолчанию. Современное ядро поддерживает множественные таблицы маршрутизации, сложные правила выбора маршрута на основе политик, а также взаимодействие с протоколами динамической маршрутизации, такими как OSPF и BGP. Понимание этих механизмов необходимо каждому, кто работает с сетевой инфраструктурой на базе Linux.

## 1. Основные стратегии маршрутизации

В Linux можно выделить три ключевые стратегии маршрутизации, которые отличаются по уровню автоматизации, сложности настройки и области применения.

### 1.1. Статическая маршрутизация

Статическая маршрутизация подразумевает ручное добавление записей в таблицу маршрутизации. Администратор явно указывает, через какой шлюз и интерфейс следует отправлять пакеты для достижения конкретной сети или хоста.

**Преимущества статической маршрутизации:**
- **Простота и предсказуемость.** Все маршруты заданы явно, их легко проверить и отладить.
- **Отсутствие служебного трафика.** В отличие от динамических протоколов, статические маршруты не требуют обмена данными между маршрутизаторами, что снижает нагрузку на сеть.
- **Высокая безопасность.** Поскольку информация о топологии не распространяется автоматически, злоумышленнику сложнее внедрить ложные маршруты.

**Недостатки:**
- **Отсутствие адаптации.** При выходе из строя канала или маршрутизатора трафик не будет автоматически перенаправлен по резервному пути.
- **Сложность масштабирования.** В больших сетях с десятками подсетей ручное управление маршрутами становится трудоёмким и чревато ошибками.

Статическая маршрутизация идеально подходит для небольших сетей, подключения к VPN-серверам, настройки шлюза по умолчанию на рабочих станциях и в случаях, когда топология сети изменяется редко.

### 1.2. Динамическая маршрутизация

Когда сеть разрастается или её топология часто меняется, на помощь приходят протоколы динамической маршрутизации. Маршрутизаторы автоматически обмениваются информацией о доступных сетях, строят карту сети и самостоятельно выбирают оптимальные пути. В Linux поддержка динамической маршрутизации реализуется через специальные демоны, которые взаимодействуют с ядром.

Рассмотрим три наиболее распространённых протокола.

#### RIP (Routing Information Protocol)

RIP — один из старейших дистанционно-векторных протоколов. Он основан на алгоритме Беллмана — Форда и использует количество переходов (hop count) в качестве метрики. Максимальное число переходов ограничено 15, что делает RIP непригодным для крупных сетей.

**Особенности RIP:**
- Прост в настройке и поддержке.
- Периодически (каждые 30 секунд) рассылает всю таблицу маршрутизации соседям, что создаёт заметный служебный трафик.
- Медленная сходимость при изменениях в сети.

Несмотря на возраст, RIP до сих пор применяется в небольших сетях и для обучения основам маршрутизации.

#### OSPF (Open Shortest Path First)

OSPF — современный протокол состояния каналов (link-state). Каждый маршрутизатор строит полную карту сети, вычисляя кратчайший путь с помощью алгоритма Дейкстры. OSPF лишён многих ограничений RIP.

**Ключевые преимущества OSPF:**
- Быстрая сходимость: изменения топологии распространяются практически мгновенно.
- Поддержка бесклассовой адресации (CIDR) и VLSM.
- Возможность разделения сети на области (areas) для снижения нагрузки на маршрутизаторы.
- Использование multicast-рассылки обновлений, что снижает широковещательный трафик.

OSPF широко применяется в корпоративных сетях среднего и крупного размера.

#### BGP (Border Gateway Protocol)

BGP — протокол маршрутизации между автономными системами (Autonomous Systems, AS). Именно BGP лежит в основе глобальной маршрутизации в Интернете. В отличие от внутренних протоколов (IGP), таких как RIP и OSPF, BGP оперирует не скоростью, а политиками (policy-based routing). Администратор может управлять тем, каким путём трафик покидает его сеть и входит в неё.

**Основные черты BGP:**
- Работает поверх TCP (порт 179), что обеспечивает надёжную доставку обновлений.
- Поддерживает огромные таблицы маршрутизации (в глобальном Интернете их размер превышает 900 тысяч записей).
- Использует атрибуты маршрутов (AS_PATH, LOCAL_PREF, MED) для принятия решений о выборе пути.

BGP незаменим для интернет-провайдеров, дата-центров и крупных организаций, имеющих несколько подключений к разным операторам связи.

### 1.3. Маршрутизация на основе политик (Policy Routing)

Традиционная маршрутизация принимает решение исключительно на основе IP-адреса получателя. Policy Routing позволяет учитывать и другие параметры пакета, что даёт невероятную гибкость в управлении трафиком.

**Критерии, по которым может выбираться маршрут:**
- IP-адрес источника;
- Входящий сетевой интерфейс;
- Протокол транспортного уровня (TCP или UDP);
- Номера портов источника или назначения;
- Маркировка пакета (fwmark), установленная файрволом (iptables/nftables).

**Механизм реализации:**
В Linux существует несколько независимых таблиц маршрутизации (по умолчанию используется `main`). С помощью команды `ip rule` создаются правила, которые определяют, какую таблицу использовать для конкретного пакета.

**Пример настройки policy routing:**
Предположим, у сервера два интернет-канала: быстрый, но с дорогим трафиком, и медленный, но дешёвый. Мы хотим, чтобы SSH-трафик (порт 22) всегда ходил через дорогой канал, а весь остальной — через дешёвый.

1. Создаём отдельную таблицу маршрутизации (например, с идентификатором 100).
2. Добавляем правило: `ip rule add fwmark 1 table 100`
3. С помощью iptables помечаем все пакеты с портом назначения 22 меткой 1.
4. В таблице 100 прописываем маршрут по умолчанию через шлюз дорогого провайдера.

Policy Routing активно используется для балансировки нагрузки, резервирования каналов, организации VPN-туннелей и реализации сложных схем доступа.

## 2. Инструменты управления маршрутизацией в Linux

Исторически в Linux существовало два поколения инструментов для работы с сетью.

### 2.1. Устаревший пакет `net-tools`

Включает команды `ifconfig`, `route`, `netstat`. Они знакомы многим администраторам со стажем, однако на сегодняшний день считаются устаревшими. Основные недостатки:
- Не поддерживают многие современные возможности ядра Linux (например, несколько таблиц маршрутизации, политики).
- Интерфейс команд не унифицирован.
- Разработка и поддержка пакета практически остановлены.

### 2.2. Современный пакет `iproute2`

Это стандарт де-факто для всех современных дистрибутивов Linux. Основная утилита — `ip` — предоставляет единый интерфейс для управления сетевыми интерфейсами, адресами, маршрутами, ARP-таблицей, туннелями и многим другим.

**Ключевые подкоманды `ip`:**
- `ip link` — управление сетевыми интерфейсами (включение/выключение, изменение MTU).
- `ip addr` — назначение и просмотр IP-адресов.
- `ip route` — просмотр и изменение таблиц маршрутизации.
- `ip rule` — управление правилами политики маршрутизации.
- `ip neigh` — работа с ARP-таблицей.

## 3. Linux как полноценный маршрутизатор

Любой компьютер с установленным Linux может выполнять функции аппаратного маршрутизатора. Для этого необходимо выполнить два основных шага:

1. **Включить IP-форвардинг** — разрешить ядру пересылать пакеты между интерфейсами. Это делается записью `1` в файл `/proc/sys/net/ipv4/ip_forward` или постоянной настройкой в `/etc/sysctl.conf`:

2. **Установить и настроить демон динамической маршрутизации.** Наиболее популярными являются BIRD и FRRouting.

### 3.1. BIRD (BIRD Internet Routing Daemon)

BIRD был разработан как академический проект в Карловом университете в Праге и со временем превратился в один из самых производительных и гибких демонов маршрутизации. Он поддерживает протоколы RIP, OSPF, BGP, Babel и другие.

**Особенности BIRD:**
- Высокая скорость обработки обновлений, что критично для точек обмена трафиком (IXP).
- Гибкий язык конфигурации, позволяющий реализовывать сложные политики фильтрации и манипуляции с атрибутами BGP.
- Эффективное использование памяти и процессора.
- Поддержка IPv4 и IPv6.

BIRD широко используется крупными интернет-компаниями и точками обмена трафиком по всему миру.

### 3.2. FRRouting (FRR)

FRRouting — это форк проекта Quagga, который, в свою очередь, был форком Zebra. FRR на сегодняшний день является самым функциональным набором демонов маршрутизации для Linux. Он реализует поддержку множества протоколов: BGP, OSPFv2/v3, RIP, ISIS, PIM (multicast), LDP и других.

**Ключевые преимущества FRR:**
- Модульная архитектура: каждый протокол работает в отдельном демоне (`bgpd`, `ospfd`, `ripd` и т.д.), а центральный демон `zebra` взаимодействует с ядром.
- Наличие интерактивной оболочки `vtysh`, интерфейс которой очень похож на Cisco IOS, что облегчает переход сетевых инженеров с оборудования Cisco.
- Активная разработка и поддержка сообществом, регулярные обновления.
- Интеграция с современными инструментами оркестрации (например, используется в проектах Cumulus Linux и SONiC).

FRRouting позволяет превратить обычный сервер Linux в мощный маршрутизатор уровня операторского класса.

## 4. Практические сценарии использования

Понимание стратегий маршрутизации позволяет решать широкий круг задач.

- **Небольшая домашняя или офисная сеть:** Настройка статического шлюза по умолчанию и, возможно, одного-двух статических маршрутов к локальным подсетям. Используется `ip route add default via ...`.
- **Выход в Интернет через несколько провайдеров:** С помощью policy routing можно организовать резервирование каналов (если один провайдер упал, трафик идёт через другого) или балансировку исходящего трафика.
- **VPN-сервер:** Статические маршруты направляют трафик в туннельные интерфейсы для доступа к удалённым сетям.
- **Внутренняя маршрутизация в дата-центре:** Использование OSPF для автоматического построения маршрутов между серверами и коммутаторами.
- **Участие в глобальной маршрутизации:** Сервер с BGP-демоном (BIRD или FRR) может анонсировать собственные IP-адреса нескольким провайдерам, обеспечивая отказоустойчивость и оптимизацию входящего трафика.

## Заключение

В ходе доклада мы рассмотрели три основные стратегии маршрутизации в Linux: статическую, динамическую и основанную на политиках. Мы познакомились с протоколами RIP, OSPF и BGP, каждый из которых занимает свою нишу в современной сетевой инфраструктуре. Также были изучены инструменты управления — от устаревшего `net-tools` до актуального `iproute2`, и программные реализации маршрутизаторов на базе Linux — BIRD и FRRouting.

Linux предоставляет исключительно богатые возможности для построения сетевых решений любой сложности. Гибкость, производительность и открытость делают его серьёзным конкурентом проприетарному оборудованию. Знание принципов маршрутизации в UNIX-подобных системах — обязательный навык для современного системного администратора, DevOps-инженера или сетевого архитектора.
