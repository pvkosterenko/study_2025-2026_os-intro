Введение

Linux — одна из ведущих серверных платформ. Маршрутизация в Linux позволяет гибко управлять потоками данных, превращая обычный компьютер в мощный сетевой маршрутизатор.

В докладе рассматриваются:

  • Стратегии маршрутизации (статическая, динамическая, политики)
  • Протоколы (RIP, OSPF, BGP)
  • Инструменты управления (iproute2)
  • Программные маршрутизаторы (BIRD, FRRouting)

Стратегии маршрутизации в Linux

  • Статическая маршрутизация
  • Динамическая маршрутизация
  • Маршрутизация на основе политик (Policy Routing)

Статическая маршрутизация

Маршруты добавляются вручную администратором.

Преимущества: простота, предсказуемость, нет служебного трафика.

Недостатки: нет адаптации при сбоях, сложность масштабирования.

ip route add 192.168.2.0/24 via 192.168.1.1 dev eth0

Применяется в небольших сетях и для подключения к VPN.

Динамическая маршрутизация

Маршрутизаторы обмениваются информацией автоматически, адаптируясь к изменениям.

Основные протоколы: RIP, OSPF, BGP.

RIP (Routing Information Protocol)

  • Дистанционно-векторный протокол
  • Метрика — количество переходов (макс. 15)
  • Простая настройка, но медленная сходимость
  • Подходит для малых сетей

OSPF (Open Shortest Path First)

  • Протокол состояния каналов (link-state)
  • Быстрая сходимость, поддержка CIDR
  • Разделение на области (areas)
  • Используется в корпоративных сетях

BGP (Border Gateway Protocol)

  • Маршрутизация между автономными системами (AS)
  • Основа Интернета
  • Использует политики (атрибуты AS_PATH, LOCAL_PREF и др.)
  • Работает поверх TCP (порт 179)

Маршрутизация на основе политик

Решение принимается не только по IP назначения, но и по:

  • IP источника
  • Входящему интерфейсу
  • Портам, протоколу
  • Маркировке пакета (fwmark)

Реализуется через несколько таблиц маршрутизации и команду ip rule.

ip rule add fwmark 1 table 100

Инструменты управления маршрутизацией

Устаревший пакет net-tools

  • ifconfig, route, netstat
  • Не поддерживает современные функции ядра
  • Не рекомендуется к использованию
route add -net 10.0.0.0 netmask 255.0.0.0 gw 192.168.1.1

Современный пакет iproute2

  • Стандарт де-факто для Linux
  • Единая утилита ip
  • Поддержка множества таблиц и политик
ip route show
ip route add 172.16.0.0/12 via 10.0.0.1
ip route add default via 192.168.1.1

Linux как маршрутизатор

Для превращения Linux в маршрутизатор необходимо:

  1. Включить IP-форвардинг: net.ipv4.ip_forward = 1
  2. Установить демон динамической маршрутизации (BIRD или FRRouting)

BIRD Internet Routing Daemon

  • Высокая производительность, поддержка RIP, OSPF, BGP
  • Гибкий язык конфигурации
  • Используется на точках обмена трафиком (IXP)

FRRouting (FRR)

  • Форк Quagga, наиболее функциональный
  • Модульная архитектура (zebra, bgpd, ospfd)
  • Интерфейс vtysh в стиле Cisco
  • Поддержка многих протоколов: BGP, OSPF, ISIS, PIM

Практические сценарии

  • Домашняя/офисная сеть: статический шлюз
  • Мультихоминг: резервирование и балансировка через policy routing
  • VPN: маршруты в туннель
  • Дата-центр: OSPF для автоматического построения маршрутов
  • Глобальная BGP-маршрутизация с анонсированием собственных префиксов

Заключение

  • Linux предоставляет три стратегии: статическая, динамическая, политики
  • Протоколы RIP, OSPF, BGP покрывают разные масштабы сетей
  • Инструмент iproute2 — современный стандарт управления
  • BIRD и FRRouting превращают Linux в полноценный маршрутизатор

Глубокое понимание этих механизмов необходимо системным администраторам и сетевым инженерам.
